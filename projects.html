⚠ Extreme Projects

Warning: these are not weekend projects. Each one takes weeks to months and will break your brain at least once. They're here because finishing any one of them will teach you more than years of tutorials — and because very few developers ever attempt them.
VERSION CONTROL Legendary
Build Git from Scratch
Implement the core Git internals: content-addressable object store, blob/tree/commit objects, the index (staging area), branches, HEAD, and basic merge. Not a wrapper around Git — a real implementation. When it can init, add, commit, branch, and merge, you're done.
Steps
  1. Implement the object store: blobs, trees, commits as SHA-1 hashed files
  2. Build the index file (.git/index) to track staged changes
  3. Implement refs: branches, HEAD, tag pointers
  4. Write the diff engine — compare tree objects recursively
  5. Implement 3-way merge and conflict detection
You'll learn
  • How content-addressable storage actually works
  • Why Git history is a linked list of immutable snapshots
  • What a merge conflict really is at the data level
  • Why Git is so fast despite tracking everything
SHA-1 hashing file I/O DAGs diff algorithms Python / C / Go

Implement container isolation using Linux namespaces (pid, net, mnt, uts, ipc), cgroups for resource limits, and an overlay filesystem for layered images — then run a real process inside it. This is what docker run actually does.

You'll learn

Build a Linker
Take compiled .o object files (ELF format) and link them into a working executable binary — resolving symbols, performing relocations, laying out sections, and writing a valid ELF header. Run the output directly. Most developers never look at what happens between compilation and execution.
Steps
  1. Parse ELF object files: read section headers and symbol tables
  2. Build a global symbol table and resolve all undefined symbols
  3. Merge input sections (.text, .data, .bss) into output sections
  4. Apply relocations: patch addresses now that layout is known
  5. Write a valid ELF executable and run it
You'll learn
  • What happens between the compiler and the running program
  • How symbol resolution and undefined references actually work
  • What "relocation" means and why it's necessary
  • How the OS loads an ELF binary into memory
C / Python ELF format symbol tables relocations section layout

COMPILERS Legendary
Compile Your Language to Native Code
Take your interpreted language and add a compilation backend that emits real x86-64 assembly or LLVM IR. A tree-walking interpreter is a warm-up; code generation requires understanding register allocation, calling conventions, and the ABI. The gap between these two is enormous.
Steps
  1. Add a type-checking pass over the AST
  2. Lower the AST to a simple IR (three-address code)
  3. Implement naive register allocation (spill everything to the stack first)
  4. Emit x86-64 assembly following the System V ABI
  5. Link with the C runtime and run the generated binary
You'll learn
  • What compilers actually do between source code and machine code
  • How calling conventions and stack frames work at the assembly level
  • Why register allocation is NP-complete (and how to cheat)
  • Why compiled languages are faster than interpreted ones
x86-64 assembly LLVM IR register allocation System V ABI IR lowering

COMPILERS / VMs Legendary
Build a Bytecode Virtual Machine
Design your own bytecode instruction set, write a compiler that targets it, and build a stack-based or register-based VM that executes it. This is how Python, Java, Ruby, and Lua actually run. Much faster than tree-walking, teaches a completely different set of skills.
Steps
  1. Design the bytecode instruction set (opcodes, operands, encoding)
  2. Write a compiler from your AST to bytecode chunks
  3. Build the VM: a main loop with an instruction pointer and value stack
  4. Implement function calls with call frames on the stack
  5. Add a simple mark-and-sweep garbage collector
You'll learn
  • How Python's CPython and the JVM actually execute code
  • Why bytecode is a good middle ground between interpreter and compiler
  • How call frames and the value stack model function calls
  • How garbage collectors know what memory is still in use
bytecode design stack VM call frames mark-and-sweep GC C / Rust

Implement the Raft Consensus Algorithm
Build a cluster of nodes that agree on a single ordered log of entries even when nodes crash and messages are delayed or dropped. Implement Raft fully: leader election, log replication, safety under network partitions, and snapshotting. Then write a chaos test that kills nodes randomly.
Steps
  1. Implement leader election with randomised election timeouts
  2. Implement AppendEntries: replicate log entries from leader to followers
  3. Ensure safety: a leader only commits entries replicated to a majority
  4. Add log snapshotting so nodes can compact old entries
  5. Write a chaos test that partitions and kills nodes and verifies consistency
You'll learn
  • Why distributed consensus is one of the hardest problems in CS
  • How etcd, CockroachDB, and Consul achieve strong consistency
  • What split-brain means and why it's catastrophic
  • How to reason about distributed systems using formal invariants

Build a BitTorrent Client

Steps

  2. Contact a tracker over HTTP and parse the peer list
  3. Implement the BitTorrent peer handshake and message protocol
  4. Request, download, and SHA-1 verify pieces from multiple peers

Build a Ray Tracer
Render photorealistic images from scratch: fire rays from a virtual camera, compute ray-sphere and ray-triangle intersections, model diffuse and specular reflections, implement shadows, reflections, refractions, and soft shadows. No GPU, no library — just maths and pixels in a file.
Steps

You'll learn

  • How light physically interacts with different material types

Build a Neural Network from Scratch
Implement a multi-layer neural network in raw NumPy or pure code — no PyTorch, no TensorFlow. Implement forward propagation, backpropagation through the chain rule, gradient descent, and common activations. Train it to recognise handwritten digits (MNIST) from scratch.
Steps
  1. Implement a Layer class with weights, biases, and an activation function
  2. Write the forward pass: compute activations through every layer
  3. Implement backpropagation: compute gradients via the chain rule
  4. Implement mini-batch SGD and train on MNIST
  5. Reach 97%+ accuracy, then visualise what each neuron activates on
You'll learn
  • How neural networks actually learn — not the metaphor, the maths
  • What backpropagation is doing at every step through the chain rule
  • Why PyTorch and TensorFlow exist (gradient tracking is hard)
  • How activation functions and layer depth affect what a network can learn
